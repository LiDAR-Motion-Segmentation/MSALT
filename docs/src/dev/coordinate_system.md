# Coordinate Systems & Conventions

MSALT operates across several different 2D and 3D coordinate spaces. To successfully project data between LiDAR sensors, cameras, and the computer screen, the system strictly adheres to the following coordinate conventions.



## 1. World / LiDAR Space (3D)
The global reference frame for MSALT is the LiDAR sensor's coordinate system, following standard NuScenes and right-hand rule conventions.

* **Origin (0, 0, 0):** The exact physical center of the LiDAR sensor on the roof of the ego-vehicle.
* **X-Axis:** Points **Forward** (out the front windshield of the car).
* **Y-Axis:** Points **Left** (out the driver's side window).
* **Z-Axis:** Points **Up** (towards the sky).
* **Units:** Meters.

*Note: Because the sensor is on the roof, the ground plane typically exists at a negative Z value (e.g., Z approx -1.5m).*

## 2. Camera Space (3D)
When calculating Extrinsic matrix transformations, MSALT converts LiDAR points into the local 3D coordinate system of the specific camera lens. This follows the standard OpenCV convention.



* **Origin (0, 0, 0):** The optical center of the camera lens.
* **Z-Axis:** Points **Forward** (representing the depth into the scene).
* **X-Axis:** Points **Right**.
* **Y-Axis:** Points **Down**.
* **Units:** Meters.

## 3. Image / Pixel Space (2D)
When 3D points are projected through the Intrinsic matrix (K), they land on the 2D image plane.

* **Origin (0, 0):** The **Top-Left** corner of the image array.
* **U (or X):** Pixel column, increasing to the Right.
* **V (or Y):** Pixel row, increasing Downwards.
* **Units:** Pixels.

## 4. Bounding Box (OBB) Conventions
All 3D Oriented Bounding Boxes generated by AI or manually drawn by users are stored as `BoundingBox3D` objects with the following properties:

* **Position (X, Y, Z):** The geometric center of the box in the LiDAR World Space.
* **Dimensions (dx, dy, dz):** * `dx` (Length): The size of the box along its primary heading axis.
  * `dy` (Width): The size of the box perpendicular to its heading.
  * `dz` (Height): The vertical size of the box.
* **Heading (Yaw):** The rotation of the box around the Z-axis, measured in **Radians**. An angle of 0 means the box is facing perfectly straight along the global +X axis.

## 5. UI & OpenGL Normalized Device Coordinates (NDC)
When interacting with the 3D LiDAR viewport, MSALT must translate your 2D mouse clicks into 3D raycasts. 

PyQt places the `(0,0)` mouse origin at the Top-Left of the widget. However, OpenGL requires Normalized Device Coordinates (NDC) where the origin is the absolute center of the screen, and the $Y$-axis points Up. 

MSALT performs this conversion under the hood using the following mathematical mapping:

$$ndc_x = \left(\frac{2.0 \cdot x_{mouse}}{width}\right) - 1.0$$
$$ndc_y = 1.0 - \left(\frac{2.0 \cdot y_{mouse}}{height}\right)$$

This ensures that regardless of how you resize the MSALT application window, your mouse clicks are mathematically accurate.
